# Windows EXE 打包说明

由于你目前没有 Windows 环境，我提供了以下两种方案来获取打包好的 exe 文件：

---

## 方案一：使用 GitHub Actions 自动打包（推荐）

我已为你配置好 GitHub Actions，只需将代码推送到 GitHub，就会自动打包并生成可下载的 exe 文件。

### 操作步骤

1. **创建 GitHub 仓库**
   - 访问 https://github.com/new
   - 创建一个新的仓库（如 `excel-desensitizer`）

2. **推送代码到 GitHub**
   ```bash
   cd excel-data-desensitization

   # 初始化 git 仓库
   git init

   # 添加所有文件
   git add .

   # 提交
   git commit -m "Initial commit"

   # 添加远程仓库（替换为你的仓库地址）
   git remote add origin https://github.com/你的用户名/excel-desensitizer.git

   # 推送
   git push -u origin main
   ```

3. **下载打包好的 exe**
   - 推送后，访问 GitHub 仓库的 Actions 页面
   - 等待工作流运行完成（约 2-3 分钟）
   - 在最新的工作流运行结果中，点击 "Artifacts" 下载 exe 文件

4. **或者从 Release 页面下载**
   - 推送后，会自动创建一个 Release
   - 访问仓库的 Releases 页面
   - 下载最新的 exe 文件

## 方案二：使用 Docker 打包（如果你有 Docker）

如果你有 Docker 环境，可以使用 Windows 容器来打包：

```bash
# 运行 Windows 容器进行打包
docker run -it --rm \
  -v "$(pwd):/app" \
  -w /app \
  python:3.11-windowsservercore \
  powershell -Command "
    pip install pandas openpyxl Flask Werkzeug pyinstaller;
    pyinstaller --clean --noconfirm pyinstaller.spec
  "
```

> 注意：这需要 Docker Desktop 并启用 Windows 容器模式

## 方案三：找一台 Windows 电脑临时打包

如果有朋友或同事的 Windows 电脑，可以临时借用：

1. 将整个项目文件夹复制到 Windows 电脑
2. 安装 Python（https://python.org）
3. 打开命令提示符，执行：
   ```cmd
   cd excel-data-desensitization
   pip install pandas openpyxl Flask Werkzeug pyinstaller
   python build.py
   ```
4. 打包完成后，`dist/` 目录下就是 exe 文件

## 方案四：使用在线 CI 服务

如果不想用 GitHub Actions，也可以使用其他 CI 服务：

- **Azure Pipelines**: 免费支持 Windows 构建
- **AppVeyor**: 专为 Windows 构建设计
- **CircleCI**: 支持 Windows 执行环境

这些服务都提供免费的 Windows 构建时长，使用方法类似 GitHub Actions。

## 打包后的使用说明

获取到 `Excel数据脱敏工具.exe` 后：

1. **双击运行**，程序会自动在后台启动
2. **自动打开浏览器**（约等待 1-2 秒）
3. 如果浏览器没有自动打开，手动访问 `http://127.0.0.1:端口号`
4. 使用完成后，**关闭浏览器**并在任务栏右键退出程序

## 常见问题

### Q: 运行时提示缺少 DLL？
A: 如果目标 Windows 电脑缺少运行时库，可能需要安装 [Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)

### Q: 杀毒软件报毒？
A: PyInstaller 打包的程序有时会被误报，可以：
- 将 exe 添加到杀毒软件白名单
- 或者使用 `--onefile` 以外的打包方式

### Q: 文件太大？
A: 由于包含了 Python 解释器和所有依赖，exe 文件约 30-50MB，这是正常的。
